Frontend stack
•  Framework: React
•  Styling: Tailwind CSS
•  Realtime transport to UI: SSE (Server-Sent Events) from backend
•  Goal: “War-room” dashboard showing agent reasoning live

UI design (visual language)
•  Theme: dark operations dashboard
•  Main visualization: agent pipeline / system map
•  Color semantics in diagram:
◦  Purple: control (Orchestrator)
◦  Blue: detection (Watcher)
◦  Amber: analysis (Diagnoser)
◦  Teal: action/fix (Patcher)
◦  Coral: communications (Communicator)
◦  Green: resolved/report state
◦  Gray: shared infra (event bus, external systems)

Core frontend features to build (MVP)
1. Trigger control
◦  Button to fire a fake incident (example: CPU spike on api-server-2)
2. Live agent pipeline visualizer
◦  Show 5 agents and activation sequence
3. Streaming “thought/output” feed
◦  Real-time bubbles/log entries per agent as each finishes
4. Incident status card
◦  Current stage, severity/context, active agent
5. Final report view
◦  Structured diagnosis + suggested patch + comms draft

UX interaction flow
1. User clicks “simulate incident”
2. UI enters incident-active state
3. Agents run with ~1–2s visible progression each
4. Outputs stream into feed in order
5. UI ends on “incident resolved + report ready”

Demo-ready frontend expectations
•  Show clear sequential flow of context between agents
•  Keep reasoning visible (high demo impact)
•  Support 2–3 scenario demos (from mock incidents)

“Everything else” relevant to frontend
•  Backend is expected to orchestrate and stream events; frontend consumes those events and renders state transitions.
•  For later rounds, frontend expansion includes:
◦  Incident history dashboard (in-person roadmap)
◦  Potential richer dependency/SLA/audit visualizations (final round roadmap)

Your markdown says agents pass output sequentially (no shared global state), while the diagram shows a shared event bus. For implementation clarity, pick one narrative and keep UI labels consistent with it.
